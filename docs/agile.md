# Agile

Agile es una forma de gestionar proyectos de software. Busca maximizar el valor de lo que se le entrega al cliente, reducen riesgos y aceleran el timepo en el que se piensa una idea, y esta se comercializa.

# Scrum

Es una metodologia de trabajo para los equipos autoorganizados, que busca aportarle valor al cliente en un cierto periodo de tiempo (Sprint).

## Roles 
- Product Owner: prioriza qué se debe hacer y gestiona el Product Backlog.
- Scrum Master: ayuda al equipo a seguir Scrum y elimina impedimentos.
- Equipo de desarrollo: Es el equipo que se encarga de desarrollar, pruebar y entregar el producto.

## Elementos clave
- Product Backlog: lista con priotidades de lo que se busca en el producto.
- Sprint: Ciclo de tiempo donde el equipo trabaja para crear un aumento de producto finalizado
- Sprint Backlog: Tareas para un sprint en concreto.
- Sprint Review: Revision del sprint, se hace una vez este se ha finalziado
- Sprint Retrospective: Reunion del equipo para mejorar la forma de trbajo

# Kanban 
Kanban es una forma de gestionar el trabajo de forma visual con una tabla, que en cada columna muestra un estado. Cada tarea se va moviendo por las columnas segun vaya cambiando su estado.

# Diferencias entre Scum y Kanban 
- Tiempo:
 - Scrum usa sprints de cierta suracion determinada.
 - Kanban trabaja con un flujo continuo.
- Roles:
 - Scrum define roles especificos.
 - Kanban no exige roles nuevos porque se pueden aplicar sobre una estructura.
- Planificacion:
 - Scrum, se planifica al comienzo de cada sprint.
 - Kanban, se gestionan las tareas de forma continua.

# Cuando usar cada metodologia

- Scrum es una mejor opcion cuando el producto va evolucionando en distintas versiones y se debe establecer una estructura clara de roles
- Kanban es mas conveniente cuando el trabajo cambia constantemente, por ejemplo soporte o mantenimiento.

Pero al final en la practica, se pueden combinar las 2 metodologias.
